Project #1 — Crypto Mixer De-Anonymization & Cross-Chain Laundering Tracer

12-Week Intern Program · Summer 2026 · Confidential

Part A — Program Brief

What This Is

A 12-week intern program to build a platform that traces criminal money through crypto mixers and across multiple blockchains — even when bad actors deliberately obscure the trail. The end product is sellable to law enforcement agencies, crypto compliance teams, and exchanges.

Intern Ownership

Three interns will own the build end-to-end.

  • Intern A — Data collection and cross-chain tracing
  • Intern B — Pattern detection and the wallet relationship map
  • Intern C — Nation-state flagging and the investigation dashboard

Program Structure

The first two weeks are dedicated to learning — blockchain fundamentals, version control, and hands-on API practice. Building begins in Week 3. The core platform is expected to be largely complete by Weeks 10–11, with Week 12 reserved for finishing touches and handoff.

Success Criterion

Platform traces a test transaction from Bitcoin through a mixer and across two blockchains to its final destination — verified live, in front of a non-technical stakeholder.

Commercial Angle

Sellable service to law enforcement agencies, crypto compliance teams, and exchanges.

Standards

  • Weekly standard: Something real ships every week. A week where nothing usable is produced is a week to pay attention to.
  • Never: Passwords or credentials stored in code. All scores labeled as probabilistic — never definitive conclusions.

Part B — The Six Modules

M1 — Blockchain Data Collection

Foundation everything else depends on. Pulls raw transaction data from Bitcoin, Ethereum, Tron, and Solana. All data stored locally so the platform keeps working even if a data source goes down. Collection runs on schedule — no manual triggering.

Definition of Done
  • Enter any wallet address — full transaction history appears within 30 seconds
  • Works for Bitcoin, Ethereum, Tron, and Solana
  • Data source outages handled gracefully — no crashes
  • All four networks collecting data simultaneously
Acceptance Test: Enter a known public wallet address; confirm full transaction history appears in dashboard within 30 seconds.

M2 — Mixer Analysis Engine

Detects patterns revealing which transactions passed through the same mixing round — similar timing, similar output amounts, address behavior relative to one another. Every finding labeled with a confidence score — nothing presented as definitive.

Definition of Done
  • Given a post-mixer wallet, identify at least 2 others from the same mixing round
  • Timing patterns shown visually on timeline
  • Amount distribution report shows how funds were broken up before and after mixing
  • Confidence score displayed for each wallet link
Acceptance Test: Run analysis on a known public mixer case; confirm system identifies the mixing round and links correct addresses.

M3 — Cross-Chain Tracer

Follows money across blockchains through bridges and exchanges. Maintains a list of known transfer points. Links the sending wallet on one network to the receiving wallet on another.

Definition of Done
  • Follow funds from Bitcoin to Ethereum to Tron automatically
  • Each transfer point labeled with bridge or exchange name
  • Full path displayed visually: Network A Wallet → Transfer Point → Network B Wallet
  • Top 10 known transfer points in database
Acceptance Test: Trace a known cross-chain transaction from a public hack report; confirm transfer point and destination wallet correctly identified.

M4 — Wallet Relationship Map

Stores every wallet and transaction as a connected network. Known entities labeled — exchanges, mixers, criminal groups. Analysts can query: "show me everything connected to this wallet within 3 steps."

Definition of Done
  • Every traced wallet stored with attributes and connections
  • Wallet groups shown as connected clusters with group identifier
  • Known entities labeled
  • Searchable: "show me all wallets connected to X within 3 steps"
Acceptance Test: Search a known exchange deposit address; confirm map shows connected wallets and correct entity label.

M5 — Nation-State Pattern Flagging

Loads known behavioral signatures from public reports (Lazarus Group, etc.). Every wallet cluster receives a 0–100 likelihood score. Scores above 70 trigger a dashboard alert. Every score comes with a plain-English explanation.

Definition of Done
  • Every analyzed wallet cluster receives a 0–100 likelihood score
  • Scores above 70 trigger alert in dashboard
  • Each score explained with specific patterns that contributed
  • Known state-sponsored wallets score above 80 in testing
Acceptance Test: Run a known state-sponsored wallet cluster; confirm score above 70 with plain-English explanation.

M6 — Investigation Dashboard

The interface analysts actually use, runs in any browser. Enter a wallet address and get a complete forward and backward money trail. High-scoring wallets surface in the alert queue. One-click PDF case report export.

Definition of Done
  • Interactive wallet network map: zoom, filter by network, click for details
  • Trace view: enter address, see full forward and backward money path
  • Alert queue showing flagged wallets with scores and explanation
  • One-click PDF case report export
Acceptance Test: Non-technical user navigates from flagged alert to trace view and exports case report without guidance.
